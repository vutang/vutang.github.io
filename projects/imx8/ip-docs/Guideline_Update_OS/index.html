    Phần này trình bày tính năng cập nhật hệ điều hành

    1. Cập nhật hệ điều hành từ xa

    1. Tại giao diện, người dùng mở ứng dụng cập nhật HĐH, chọn cập nhật qua OTA
    2. Ứng dụng setup mạch để chuẩn bị cập nhật trong chế độ recovery
    3. Mạch reboot về recovery mode rồi tiến hành cập nhật
    4. Mạch reboot về normal mode

    2. Cập nhật các service của hệ thống

    3. Cập nhật HĐH tại chỗ dùng USB

    3.1. Trường hợp mạch lên U-Boot, không lên Linux

    Mạch đã xảy ra tất cả các vấn đề sau:

    • Hỏng normal OS
    • Hỏng recovery OS
    • Không boot được từ NET

    Quy trình:

    1. Người dùng cắm USB vào thiết bị (USB này do nhà phát triển thiết lập)
    2. Tắt rồi bật nguồn thiết bị
    3. Thiết bị tự động cập nhật kernel, device-tree, rootfs trong môi trường U-Boot
    4. Thiết bị tự khởi động lại vào recovery mode để đảm bảo recovery OS được cập nhật thành công
    5. Thiết bị tự khởi động về normal mode
    6. Thông báo cập nhật thành công

    3.2. Trường hợp cập nhật mới (mạch đang hoạt động bình thường – OPTIONAL)

    1. Người dùng cắm USB có chứa hệ điều hành
    2. Tại giao diện, người dùng mở ứng dụng cập nhật HĐH, chọn cập nhật qua USB
    3. Ứng dụng setup mạch để chuẩn bị cập nhật trong chế độ recovery
    4. Mạch reboot về recovery mode rồi tiến hành cập nhật
    5. Mạch reboot về normal mode

    4. Cập nhật HĐH dùng SD card (trong trường hợp mạch chết)

    Trường hợp này thường áp dụng cho mạch hỏng Boot loader

    1. Người dùng cắm SD card
    2. Tắt rồi bật nguồn thiết bị
    3. Mạch khởi động U-Boot từ SD card
    4. Tại U-boot, mạch tự động cập nhật hệ điều hành (gồm U-boot, device-tree, kernel, rootfs)
    5. Thiết bị tự khởi động lại vào recovery mode để đảm bảo recovery OS được cập nhật thành công
    6. Thiết bị tự khởi động về normal mode
    7. Thông báo cập nhật thành công

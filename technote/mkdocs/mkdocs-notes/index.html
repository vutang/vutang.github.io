    MkDocs documentation

    Using mkdocs for documentation. More detail about this tool at MKdocs

    Getting started

    • mkdocs new my-project - Create a new project

    Deploy in Github Page

    This document will be published in Github Page. To do this, you need:

    • Create a repository in github with name: vutang.github.io
    • Clone this repo to local machine
    • In local machine, use command: mkdocs gh-deploy --config-file ../mkdocs-vux-note/mkdocs.yml --remote-branch master

    Test with local webserver

    • use mkdocs serve to run local web server
    • in your browser, access http://127.0.0.1:8000

    Commands

    • mkdocs new [dir-name] - Create a new project.
    • mkdocs serve - Start the live-reloading docs server.
    • mkdocs build - Build the documentation site.
    • mkdocs help - Print this help message.

    Project layout
